---
name: Mejorar el otro software, no copiar sus problemas
description: Cuando se replica funcionalidad del otro software, mejorar la lógica, nombres y estructura — no copiar los quilombos
type: feedback
---

Cuando el usuario dice "como en el otro software", se refiere a la funcionalidad y UX, NO a copiar la implementación tal cual.

**Why:** El otro software tiene nombres confusos (cotizacion = presupuesto), tablas compartidas mezcladas, y decisiones técnicas que generaron problemas. Flux tiene arquitectura limpia y propia.

**How to apply:** Siempre repensar nombres, estados, estructuras de datos para que sean claros y directos. Usar nombres que signifiquen lo que dicen. No heredar complejidad innecesaria del otro sistema.
