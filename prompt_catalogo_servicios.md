# Análisis de descripciones de presupuestos para crear catálogo de servicios

## Contexto
Soy una empresa de herrería, reparación de portones, automatizaciones y trabajos en metal (Herreelec). Tengo un software de presupuestos donde durante mucho tiempo usé un producto genérico llamado "Servicio" (SRV-0001) para todo, pero en la descripción detallada de cada línea escribía lo que realmente era el servicio.

Ahora quiero crear un catálogo de servicios y productos organizado. Necesito que analices TODAS las descripciones detalladas de mis líneas de presupuesto y:

## Lo que necesito que hagas

1. **Identificar patrones repetidos**: Muchas descripciones se refieren al mismo servicio con redacción ligeramente diferente. Agrupá todo lo que sea el mismo servicio.

2. **Proponer un catálogo** con esta estructura para cada servicio/producto detectado:
   - **Código**: Si ya tiene uno entre corchetes (ej: [RP-PG-01]) usarlo. Si no, proponer uno coherente con la nomenclatura existente.
   - **Nombre corto**: Nombre comercial limpio (ej: "Cambio de zócalo inferior")
   - **Tipo**: "servicio" o "producto"
   - **Categoría sugerida**: Agrupar en categorías lógicas (ej: "Reparación de portones generales", "Portones levadizos", "Automatización", "Logística", "Fabricación a medida", etc.)
   - **Descripción de venta**: Un texto estándar que se pueda poner en los presupuestos
   - **Precio sugerido**: Basado en el promedio de uso, o rango si varía mucho
   - **Impuesto**: El que más se usa para ese servicio
   - **Frecuencia**: Cuántas veces se usó (dato que te paso)

3. **Nomenclatura de códigos**: Respetar el patrón existente:
   - `RP-PG-XX` = Reparación Portón General
   - `RP-PL-XX` = Reparación Portón Levadizo
   - `RP-PC-XX` = Reparación Portón Corredizo
   - `RP-PCC-XX` = Reparación Portón Curvo Corredizo
   - `TI` = Traslado e Instalación
   - `MO-MAT-XX` = Mano de obra y materiales
   - Proponer nuevos prefijos si hace falta (ej: `FAB-XX` para fabricación, `AUT-XX` para automatización)

4. **Ignorar**: Líneas que digan solo "[S] Servicio" (es el placeholder genérico), y las que sean descripciones muy específicas de un solo trabajo que no se van a repetir.

5. **Formato de salida**: Dame el resultado como una tabla o JSON que pueda importar directo a mi sistema. Campos:
   ```
   codigo, nombre, tipo, categoria, descripcion_venta, precio_sugerido, impuesto, unidad
   ```
